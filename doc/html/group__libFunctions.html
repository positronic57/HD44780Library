HD44780 supports up to 8 user defined characters. The definitions of the characters are stored in the CGRAM of the controller. CGRAM is 64 bytes long where each character is defined by 8 bytes. The function defines those special characters.

Parameters
[in]pHD44780pointer to the structur that represent HD44780
[in]charMatrixpointer to a matrix with 8 columns and up to 8 raws. Each raw holds definition of one custom character.

Init function for HD44780 controller. The function defines and configure the microcontroller pins that are used as interface for the LCD module. It performs 4-bit initializations of HD44780 and defines the number of display lines.

Parameters
[in,out]pHD44780pointer to the structur that represent HD44780
[in]HD44780_CMD_PORTis a pointer to the MCU PORT used for cmd bus
[in]HD44780_DATA_PORTis a pointer to the MCU PORT used for data bus
[in]HD44780_RSdefines the MCU pin connected to RS pin of HD44780
[in]HD44780_RSMCU defines the pin connected to RS pin of HD44780
[in]HD44780_EMCU defines the pin connected to E pin of HD44780
[in]HD44780_RWMCU defines the pin connected to RW pin of HD44780
[in]activeLinesis number of display lines. Possible values are: SINGLE_LINE_DISPLAY, TWO_LINES_DISPLAY, FOUR_LINE_DISPALY. DDRAM of single line LCD in most of the cases is divided in two 8 bytes wide memory segments. The starting addresses of those segments are defined as:
  • LCD16x1_SELECT_DDRAM_1ST_HALF;
  • LCD16x1_SELECT_DDRAM_2ND_HALF. Those LCD modules although they have a single line, due to the DDRAM layout must be configured as two line modules.
Returns
void.
